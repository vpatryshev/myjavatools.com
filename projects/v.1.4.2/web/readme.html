Running the tests for HttpClientRequest and ServerRequest

You will need to launch Tomcat, so that HttpClientTest/test1.jsp, HttpClientTest/test2.jsp, and HttpClientTest/test3.jsp are accessible from a browser.

The three test jsps contain easy instructions on how to run them – it is basically just clicking “commit” button. The test jsp then receives the form data and verifies it; in case of success it returns just one word, “Success”, otherwise there is some diagnostics there.

To run HttpClientRequest test suite, make sure that you are on the same port. Default port is 80; if your instance of Tomcat uses a different port number, you'll need either to change the number in AllTests.java, or, in case you are interested in tracing the actual data exchange, use TcpMonitor in proxy mode, sending requests to port 80 and redirecting them to whatever port you have your Tomcat running on.
