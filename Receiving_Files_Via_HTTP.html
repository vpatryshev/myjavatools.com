In my previous article I wrote about sending HTTP requests with form data, including files from Java. Now I would like to talk about the other end of the line: how to handle form data, including multipart, including files, on the server side. I will use the three examples from that article, http://www.devx.com/Java/Article/17679/0/page/1, to demonstrate my point.
Usually people would expect a servlet to process multipart requests, since it is too much for a JSP to handle. But with a properly designed decorator class for HttpServletRequest you can actually handle anything right inside your jsp.

For instance, when you, in your jsp, extract request information the first two requests, <form method="get" action="hi.iq/register.jsp"> Name: <input type="text" name="name" value="J.Doe"> email: <input type="text" name="email" value="abuse@spamcop.com"> <input type="submit"> </form>
and <form method="post" action="hi.iq/register.jsp"> Name: <input type="text" name="name" value="J.Doe"> email: <input type="text" name="email" value="abuse@spamcop.com"> <input type="submit"> </form> from that article, you do not need any decorator, since the request, represented in in your jsp by request, an instance of class javax.servlet.HttpServletRequest, handles both GET and POST, so that you can conveniently extract the values of form fields name and email using request.getParameter("name") and request.getParameter("email").
